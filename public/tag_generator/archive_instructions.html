Archive Instructions

How to archive the old Deadlock Advisor repo and hand off GitHub Pages to the new project.

Situation: The old deadlock-advisor repo hosts your current Flask + vanilla JS tag generator site on GitHub Pages. The new project will be a separate repo with the same name. These steps walk you through archiving the old one without losing any work.

Overview

  1. Rename the old repo to something like deadlock-advisor-legacy
  2. Create the new repo as deadlock-advisor
  3. The GitHub Pages URL moves automatically to the new repo
  4. Old repo stays accessible at the renamed URL for reference

Step-by-Step

1Back Up Your Old Repo (Optional but Recommended)

If you want a local backup before renaming, clone it or just verify you already have it locally:

cd path/to/old/deadlock-advisor
git status   # make sure everything is committed
2Rename the Old Repo on GitHub
  1. Go to github.com/YOUR_GITHUB_USERNAME/deadlock-advisor
  2. Click Settings (the gear icon near the top)
  3. Under "General", find the Repository name field
  4. Change it to deadlock-advisor-legacy
  5. Click Rename

GitHub will automatically redirect old links. The Pages URL YOUR_GITHUB_USERNAME.github.io/deadlock-advisor will stop serving (since the repo name changed) — this is what you want.

3Update Your Local Clone (Old Project)

If you have the old project cloned locally, update its remote URL so git doesn't get confused:

cd path/to/old/deadlock-advisor
git remote set-url origin https://github.com/YOUR_GITHUB_USERNAME/deadlock-advisor-legacy.git
4Create the New Repo

Follow Module 0 of the lesson plan: create a new repo named exactly deadlock-advisor on GitHub, then follow the setup steps to initialize the Vite project.

Once the new project is pushed and GitHub Actions deploys it, the URL YOUR_GITHUB_USERNAME.github.io/deadlock-advisor will serve the new app.

5Enable GitHub Pages on the New Repo
  1. Go to the new deadlock-advisor repo on GitHub
  2. Click Settings → Pages
  3. Under "Build and deployment", set Source to GitHub Actions
  4. Save — the next push to main will trigger the deploy workflow
6Archive the Old Repo (Optional)

If you want to mark the legacy repo as read-only and clearly outdated:

  1. Go to github.com/YOUR_GITHUB_USERNAME/deadlock-advisor-legacy → Settings
  2. Scroll to the "Danger Zone" at the bottom
  3. Click Archive this repository
  4. Confirm — the repo becomes read-only and shows an "Archived" banner

Archiving is reversible. The code and history remain fully accessible.

Data Migration

The JSON data files (heroes, items, tags) in the old project live at:

public/tag_generator/data/heroes/*.json
public/tag_generator/data/items/*.json
public/tag_generator/data/tags.json

Copy these into the new project at:

src/data/heroes/*.json
src/data/items/*.json
src/data/tags.json

Also copy the image resources:

public/resources/  →  (new project) public/resources/
Image paths in JSON files: The image_path fields reference paths like /resources/heroes/abrams/.... These work as-is in the new project because the assetPath() helper (Module 4) prepends the correct base URL. Do not change the paths in the JSON files.

After Both Projects Are Up

  • Old project: github.com/YOUR_GITHUB_USERNAME/deadlock-advisor-legacy (archived, reference only)
  • New project: github.com/YOUR_GITHUB_USERNAME/deadlock-advisor (active, live at github.io)
  • The tag generator Flask tool still works locally — run python app.py from the old project folder as before
